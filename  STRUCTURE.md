# Структура папок проекта

Проект базируется на готовых, но кастомизируемых решениях. Проект служит внцтренней цели - наработать готовые решения для подобных ИИ-проектов.

По этим двум причинам важно следовать заданной структуре и правилам.

## src/routes

- Стандартная папка Tanstack start. Всякий файл или папка превращается в маршрут.
- Здесь держим только маршруты и код для работы с маршрутизатором. Вся логика, верстка выносится. Например, определение параметра в маршруте остается здесь, но логика использования выносится.

## src/app

- Обертка всего приложения.
- Компоненты навигации.
- Обработчик ошибок.

## src/api

- Интеграция с БД.
- Интеграция с ИИ.

## src/pages

- Для разработки поведения и верстки страниц.
- Одна папка - одна страница.

## src/components

- Внешние компоненты Vercel - shadcn и ai-elements.
- Редактировать запрещено для четкого разделения внешней зависимости.
- Название не логично, т.к. его занимает библиотека, а настроить не удалось.

## src/elements

### src/elements/reusables

- Переиспользуемые компоненты. Это либо кастомизация для src/components, либо компоненты, которые явно остануться только в этом проекте.
- Функциональны, т.е. не должны быть привязаны к морфологии проекта.

### src/elements/composables

- Переиспользуемые компоненты. Используют reusables.
- В отличии от reusables, связаны морфологией.

## src/kit

- Компоненты, которые сразу претендуют быть частью Rolder UI Kit v3.

## Остальное

- `src/lib` - переиспользуемы утилиты, хуки, типы.
